# openwrt-ipv6-off

**Русский** · [English](README.en.md) · [中文](README.zh-CN.md)

Менеджер отключения **IPv6** на **OpenWrt** одним скриптом: бэкап текущих настроек, отключение IPv6, **автоматический откат по IPv4** при потере связи, точная проверка статуса и диагностика. Чистый `ash`/BusyBox, без зависимостей.

Один прогон — и IPv6 выключен целиком (network / dhcp / firewall / odhcpd), а на случай, если что-то пойдёт не так, есть страховка: не поднимется IPv4-интернет — настройки вернутся сами.

## Зачем

[#зачем](#зачем)

IPv6 на роутере иногда только мешает: провайдер выдаёт кривой префикс, RA/DHCPv6 конфликтуют с локальной сетью, трафик утекает мимо прокси/VPN в обход IPv4-правил, ломается DNS. Отключить его руками — это правка нескольких секций UCI (`network`, `dhcp`, `firewall`) плюс `odhcpd`, и легко либо забыть вернуть «как было», либо отрезать себе доступ на удалённом роутере.

Скрипт снимает рутину и риск: делает всё одной кнопкой, с полным бэкапом текущей конфигурации и watchdog-страховкой, которая сама откатит изменения, если после отключения связь по IPv4 не восстановится.

## Требования

[#требования](#требования)

- OpenWrt (`ash`/BusyBox). Проверялось на fw4/nftables (22.03+) и fw3/iptables — восстановление идёт копированием сырых `/etc/config/*`, поэтому не зависит от бэкенда firewall.
- Root-доступ по SSH (или локальная консоль).
- Немного места в `/root` под бэкапы и лог (единицы КБ).

## Что делает скрипт

[#что-делает-скрипт](#что-делает-скрипт)

1. **Бэкап.** Снимает точную копию `network`, `dhcp`, `firewall` (сырые `/etc/config/*`) и сохраняет состояние `odhcpd` — чтобы откат вернул именно то, что было, без дублирования правил.
2. **Watchdog.** Запускает автономную страховку в `/tmp`: после отключения она проверяет доступ по IPv4 (пингует по очереди `1.1.1.1`, `8.8.8.8`, `77.88.8.8`, `77.88.8.1` — успех, если ответил хотя бы один) и, если связь не поднялась, сама возвращает бэкап. Работает независимо от SSH-сессии — переживает её обрыв. Адреса Яндекса в списке — подстраховка для роутеров за белыми списками, где Cloudflare/Google могут быть недоступны; список задаётся переменной `WD_PING_HOSTS` в начале скрипта.
3. **Отключение IPv6 в UCI.** Убирает `ula_prefix`, ставит `lan.ip6assign=0`, `wan.reqaddress=none` / `reqprefix=no`, удаляет `wan6`, выключает `dhcpv6`/`ra`/`ndp` на lan и wan, чистит IPv6-правила firewall (в обратном порядке индексов — иначе они сползают).
4. **odhcpd.** Останавливает и отключает автозапуск.
5. *(Опционально)* **sysctl.** Жёсткое отключение IPv6 на уровне ядра с персистентным `/etc/sysctl.d/` — включается флагом `--hard` или ответом в меню.
6. Перезапускает firewall и сеть, затем показывает точную проверку результата.

Ничего необратимого: любой запуск начинается с бэкапа, а восстановление доступно из меню, флагом `--restore-last` или вручную из `/root/ipv6-off/backups/`.

## Установка

[#установка](#установка)

Команды выполняются **на роутере** (по SSH). Установщик скачает скрипт, покажет краткую сводку (версия, размер, SHA256, что делает) и по желанию даст полистать исходник в пейджере — без вывода всего кода на экран:

```
wget -O install.sh https://raw.githubusercontent.com/lastik9/openwrt-ipv6-off/main/install.sh
sh install.sh
```

В меню установщика: `1` — установить, `2` — показать код (откроется в пейджере, выход — `q`), `3` — отмена. Пустой **Enter** ничего не устанавливает — меню просто переспросит; для установки нажмите `1`.

Без вопросов / для автоматизации:

```
sh install.sh --yes        # поставить молча
sh install.sh --yes --run  # поставить и сразу запустить менеджер
```

Если базовый `wget` из BusyBox ругается на HTTPS/SSL — замените его на `uclient-fetch`:

```
uclient-fetch -O install.sh https://raw.githubusercontent.com/lastik9/openwrt-ipv6-off/main/install.sh
```

## Удаление

[#удаление](#удаление)

Команды выполняются **на роутере** (по SSH). Деинсталлятор скачивается и запускается так:

```
wget -O uninstall.sh https://raw.githubusercontent.com/lastik9/openwrt-ipv6-off/main/uninstall.sh
sh uninstall.sh
```

Он остановит watchdog, при необходимости вернёт IPv6 из последнего бэкапа, снимет артефакт жёсткого режима (`/etc/sysctl.d/99-ipv6-off.conf`) и удалит менеджер. Бэкапы и лог по умолчанию **сохраняются** — их можно снести отдельно.

Неинтерактивно / с опциями:

```
sh uninstall.sh --yes         # без вопросов; статус IPv6 не меняется, бэкапы сохраняются
sh uninstall.sh --restore     # перед удалением вернуть IPv6 из последнего бэкапа
sh uninstall.sh --purge       # удалить всё, включая данные (бэкапы / лог / диагностику)
```

Если IPv6 сейчас **выключен**, деинсталлятор по умолчанию предложит сначала вернуть его — чтобы не остаться без интернета и без инструмента отката. Отключается это флагом `--no-restore`.

## Использование

[#использование](#использование)

Без аргументов — интерактивное меню (отключение / проверка / диагностика / лог / восстановление / управление бэкапами).

Неинтерактивно — для автоматизации и тестов:

```
sh ipv6-off.sh --disable --yes        # отключить IPv6 (бэкап + watchdog)
sh ipv6-off.sh --disable --yes --hard # то же + отключение на уровне ядра (sysctl)
sh ipv6-off.sh --restore-last --yes   # откатить последний бэкап
sh ipv6-off.sh --check                # подробная проверка; код возврата 0 = выкл, 1 = вкл
sh ipv6-off.sh --status               # печатает ON / OFF
sh ipv6-off.sh --diag                 # собрать диагностику в файл
sh ipv6-off.sh --help                 # справка
```

## Обычное отключение или жёсткое (sysctl)

[#обычное-отключение-или-жёсткое-sysctl](#обычное-отключение-или-жёсткое-sysctl)

При отключении скрипт спрашивает: `Дополнительно вырубить IPv6 на уровне ядра (sysctl)? [y/N]`. Ответ по умолчанию (просто **Enter** или `N`) — **обычное отключение**, и в большинстве случаев нужно именно оно.

**Обычное отключение (Enter / N).** IPv6 выключается через настройки OpenWrt: убираются раздача адресов, RA/DHCPv6, префиксы (`ula_prefix`, `ip6assign`, `reqaddress/reqprefix`, `wan6`), IPv6-правила firewall, останавливается `odhcpd`. Трафик по IPv6 больше не идёт и не утекает мимо прокси/VPN. При этом модуль ядра остаётся загружен, а link-local адреса (`fe80::…`) на интерфейсах сохраняются. Это чистый, обратимый способ — ровно то, что делал исходный скрипт.

**Жёсткое отключение (`y` в меню или флаг `--hard`).** Дополнительно ставит `net.ipv6.conf.{all,default,lo}.disable_ipv6=1` через персистентный файл `/etc/sysctl.d/99-ipv6-off.conf`, полностью гася стек IPv6 в ядре, включая link-local. Что это влечёт:

- Пропадают даже link-local адреса `fe80::…` на всех интерфейсах.
- Софт, который рассчитывает хотя бы на локальный IPv6, может повести себя неожиданно: отдельные демоны, иногда LuCI, межсервисное взаимодействие.
- Пользы для типовой задачи «просто выключить IPv6» это не добавляет — обычного отключения достаточно, чтобы IPv6-трафика не было.

**Вывод: жмите Enter (обычное отключение).** Режим `--hard` нужен только в специфических случаях — когда требование полностью убрать стек IPv6, или когда что-то упорно поднимает link-local вопреки UCI-настройкам и это мешает.

Откат одинаков для обоих режимов (`--restore-last` или пункт 5 меню). Если выбирали `--hard`, восстановление само удаляет `/etc/sysctl.d/99-ipv6-off.conf` и возвращает `disable_ipv6=0` — отдельно ничего чистить не нужно.

## Если после отключения пропал доступ

[#если-после-отключения-пропал-доступ](#если-после-отключения-пропал-доступ)

1. Подождите ~1 минуту — watchdog сам откатит настройки, если IPv4-интернет не поднялся.
2. Если не помогло, зайдите с LAN и выполните `sh ipv6-off.sh --restore-last --yes`.
3. В крайнем случае вручную:

```
cp -a /root/ipv6-off/backups/<последний>/{network,dhcp,firewall} /etc/config/
/etc/init.d/firewall restart && /etc/init.d/network restart
```

## Совместимость

[#совместимость](#совместимость)

Проверялось на OpenWrt с fw4/nftables (22.03+) и fw3/iptables. Восстановление firewall не зависит от бэкенда, так как настройки возвращаются копированием `/etc/config/firewall`.

Флаг `--hard` (sysctl) глушит и link-local IPv6 — используйте его осознанно, если в LAN нет IPv6-only сервисов и провайдер не завязан на IPv6.
